# BUG-002 · Navigation bar overflows viewport on mobile (360px), requiring horizontal scroll and breaking page grid

| Field | Detail |
|---|---|
| **ID** | BUG-002 |
| **Title** | Navigation bar items overflow viewport horizontally at 360px width — "Stock Footage" and "History" links are inaccessible without lateral scrolling |
| **Priority** | High |
| **Severity** | Major |
| **Status** | Open |
| **Module** | Global → Header Navigation |
| **Reported By** | Manual exploratory session / user-reported |
| **Date** | 2026-03-18 |

---

### Environment

- **Browser:** Chromium (mcp-web-inspector / Playwright automation)
- **URL:** `https://previz-engine-m1mm9ayva-valid.vercel.app/`
- **Viewport:** 360×800px (Samsung Galaxy S21 / common Android mid-range)
- **User Role:** Anonymous (no login required)
- **OS:** Windows 11

---

### Preconditions

- Browser viewport is set to 360px width (or any narrow mobile viewport)
- App is loaded on any page (Studio, Stock Footage, or History)

---

### Steps to Reproduce

1. Open `https://previz-engine-m1mm9ayva-valid.vercel.app/` in a browser at **360px viewport width**
2. Observe the header navigation bar

---

### Expected Result

At 360px viewport width, the navigation should adapt responsively. All navigation links — **Studio**, **Stock Footage**, and **History** — should be visible and accessible within the viewport without horizontal scrolling. Acceptable patterns include: a hamburger/collapsed menu, stacking nav items vertically below the logo, or reducing the nav to icon-only links.

---

### Actual Result

All navigation links remain in a single horizontal row (`flex-wrap: nowrap`) alongside the logo, causing the header to overflow the 360px viewport by **141px**:

| Nav Element | X Position | Width | Right Edge | Overflow |
|---|---|---|---|---|
| Logo (Pre-Viz Engine) | 24px | 146px | 170px | None ✓ |
| Studio | 174px | 97px | 271px | None ✓ |
| Stock Footage | 279px | 108px | **387px** | **27px outside** ✗ |
| History | 395px | 102px | **497px** | **137px outside** ✗ |

- "Stock Footage" is **partially cut off** — its right portion is invisible without scrolling
- "History" starts **35px beyond the viewport** — entirely inaccessible without horizontal scroll
- The header container shows `scrollable ↔️ 141px` — confirming 141px of unconstrained horizontal overflow
- The bug affects **all pages**: Studio, Stock Footage, and History

---

### Affected Components

| Component | Selector | Type |
|---|---|---|
| Header nav flex container | `header div.flex.items-center` | flex container |
| "Stock Footage" nav link | `a.flex.items-center >> nth=2` | anchor/nav link |
| "History" nav link | `a.flex.items-center >> nth=3` | anchor/nav link |
| Header wrapper | `header.border-b.border-border` | header |

---

### Impact Analysis

- **User impact:** On any phone with ≤375px screen width (iPhone SE, Samsung Galaxy S21, Pixel 5 — all listed in the test plan Environment Matrix), users cannot access "Stock Footage" or "History" nav links without lateral scrolling. This breaks the core navigation requirement.
- **Data/business risk:** Two of three app sections are effectively unreachable via primary navigation on the most common mobile screen sizes.
- **Violates:** US-13 Acceptance Criterion — *"Header navigation links are always visible: Studio, Stock Footage, History"*; and US-13 — *"Clicking each link routes to the corresponding page"* (links are hidden off-screen).

---

### Console / Network Evidence

No JavaScript errors or network failures observed. This is a pure CSS layout defect.

---

### Root Cause

The header flex container has **`flex-wrap: nowrap`** (computed CSS) with no responsive breakpoint or media query:

```css
/* Computed on header div.flex.items-center at 360px viewport */
display: flex;
flex-direction: row;
flex-wrap: nowrap;   /* ← root cause: never wraps */
overflow: visible;   /* ← allows content to bleed outside */
width: 312px;
```

### Traceability

- **Related User Stories:** [US-13] — *Navigate between app sections*
- **Related Test Cases:** [TC-N01] — *All header navigation links route correctly*; [TC-N02] — *Active navigation state is visually indicated*
- **Potential Duplicates:** None

---

### Suggested Fix Direction

Implement responsive navigation behaviour. Options (in order of UX quality):
1. **Hamburger menu** — collapse nav links into a toggle at breakpoints below `md` (768px)
2. **Flex wrap with logo stacking** — allow nav to wrap: logo on row 1, links on row 2
3. **Overflow hidden + scroll indicator** — last resort; hides the overflow but still requires horizontal interaction
