# BUG-001 · Error state re-appears after dismissal when clicking in/out of the prompt textarea

| Field | Detail |
|---|---|
| **ID** | BUG-001 |
| **Title** | "Video Generation Failed" error re-appears after dismissal when user focuses and blurs the prompt textarea |
| **Priority** | High |
| **Severity** | Major |
| **Status** | Open |
| **Module** | Studio → Direct Prompt → Error State |
| **Reported By** | Manual exploratory session / user-reported |
| **Date** | 2026-03-18 |

---

### Environment

- **Browser:** Chromium (mcp-web-inspector / Playwright automation)
- **URL:** `https://previz-engine-m1mm9ayva-valid.vercel.app/`
- **User Role:** Anonymous (cookie-based UUID, no login required)
- **OS:** Windows 11

---

### Preconditions

- User is on the Studio page
- A prompt that triggers a moderation block (e.g., single-character `"a"`) is ready to submit
- No prior error state is active

---

### Steps to Reproduce

1. Navigate to the Studio page (`/`)
2. Type `a` (single character) into the **"Direct Prompt for Sora"** textarea (`#direct-prompt`)
3. Click the **"Generate 3 Videos Directly"** button
4. Wait for the **"Video Generation Failed"** error state to appear (message: *"Your request was blocked by our moderation system."*)
5. Click **"Edit Prompt & Retry"** to dismiss the error and return to the form
6. Click **inside** the `#direct-prompt` textarea (focus)
7. Click **outside** the textarea — e.g., on the page heading (blur)

---

### Expected Result

The "Video Generation Failed" error state remains dismissed. The user can freely interact with the prompt textarea without the error re-appearing. The form stays in its editable state so the user can modify and resubmit their prompt.

---

### Actual Result

After clicking "Edit Prompt & Retry", the error state ("Video Generation Failed") re-appears when the user clicks in and out of the prompt textarea. The background polling loop (visible in console: `Polling statuses [Object, Object, Object]`) continues firing after dismissal. When a new poll cycle returns an error result, it resets the component error state — and the next user interaction (textarea focus/blur) triggers a re-render that surfaces the error UI again.

**Exact error message displayed (verbatim):**
> Your request was blocked by our moderation system. | Your request was blocked by our moderation system. | Your request was blocked by our moderation system.

| Component | Selector | Type |

---

### Impact Analysis

- **User impact:** Users who encounter a generation error and attempt to edit their prompt are blocked from doing so — the error state re-appears every time they interact with the prompt field, making the "Edit Prompt & Retry" flow non-functional.
- **Data/business risk:** Core Studio workflow is disrupted. Users cannot recover from errors without a full page reload, leading to loss of any in-progress prompt edits.
- **Violates:** US-01 Acceptance Criterion — *"A 'Generate Videos' button triggers the generation"* (the post-error editing flow is broken); US-01 implied requirement that users can retry after failure.

> **Note:** The polling loop continues to fire **after** the user dismisses the error via "Edit Prompt & Retry". This indicates the background status-check interval is not cancelled on error dismissal. When a subsequent poll returns a failed status, it re-sets the component error state. The next textarea interaction (focus/blur) triggers a React re-render, which surfaces the error UI.

**No failed network requests observed** at the moment of re-appearance — the error is driven by stale in-memory state from the polling loop, not a new API call.

---

### Root Cause Hypothesis

The **polling interval is not cleared** when the user clicks "Edit Prompt & Retry". The interval continues to fire, and when the next poll returns a failed video status, it updates the React state with an error object. Because the error state variable was not fully reset (or was re-set by the poll), the next user action that causes a component re-render (textarea focus/blur) re-renders the error UI instead of the form UI.

**Suggested fix direction:** Cancel/clear the polling interval and fully reset the error state when "Edit Prompt & Retry" or "Start Over" is clicked.
