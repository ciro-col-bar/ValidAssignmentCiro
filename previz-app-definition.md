# Pre-Viz Engine — App Definition Document

**Version:** v1.9.0
**URL:** https://previz-engine-m1mm9ayva-valid.vercel.app/
**Explored:** 2026-03-16

---

## 1. Overview

**Pre-Viz Engine** is an AI-powered pre-visualization tool that allows filmmakers, directors, and creative professionals to transform textual descriptions and scripts into generated videos using OpenAI's **Sora** model. The platform bridges the gap between script development and production by enabling rapid visual prototyping of scenes without a physical shoot.

**Tagline:** *"Transform your vision into pre-visualization"*

---

## 2. Core Value Proposition

| Pain Point | Solution |
|---|---|
| Pre-production visualization requires expensive shoots or storyboard artists | Generate video previews directly from text prompts |
| Translating a written script into visual shots is time-consuming | Paste a script and let AI parse and generate individual shots automatically |
| No persistent record of generated assets | History page stores all generated videos for review and reference |

---

## 3. Architecture & Technology

- **Frontend:** React SPA (Single Page Application) hosted on Vercel
- **AI Backend:** OpenAI Sora video generation model
- **User Identity:** Browser cookie-based (UUID), no authentication required
- **Navigation Model:** Single-page app with client-side routing (tabs: Studio, Stock Footage, History)

---

## 4. Navigation Structure

```
Pre-Viz Engine
├── Studio          → Direct prompt-to-video generation
├── Stock Footage   → Script parsing + batch shot generation
└── History         → View previously generated videos (own + all users)
```

## 5. Feature Breakdown

### 5.1 Studio — Direct Video Generation

**Purpose:** Generate AI videos by writing a direct prompt straight to Sora.

**Key UI Elements:**

| Element | Type | Description |
|---|---|---|
| Direct Prompt for Sora | Textarea (`#direct-prompt`) | Free-text prompt sent verbatim to Sora |
| Number of Videos to Generate | Number input (`#num-videos`) | 1–5 simultaneous videos (default: 3) |
| Video Duration | Select | 4s / 8s / 12s |
| Aspect Ratio | Select | Landscape / Portrait |
| High Quality Mode | Toggle | Slower generation; higher fidelity output |
| Generate Videos | Button | Triggers batch video generation |

**Behavior Notes:**
- Prompt is passed **unmodified** to Sora — no AI enhancement or paraphrasing
- Multiple videos (1–5) can be generated simultaneously from the same prompt
- The hint text encourages specificity: *"camera movements, lighting, pacing, and technical details"*

---

### 5.2 Stock Footage Generator — Script-to-Shots

**Purpose:** Parse a full film/video script and automatically extract individual shots, then generate a video for each shot.

**Key UI Elements:**

| Element | Type | Description |
|---|---|---|
| Script Text | Textarea (`#script-text`) | Full script pasted by the user (large input area) |
| Video Duration | Select | 4s / 8s / 12s |
| Aspect Ratio | Select | Portrait / Landscape |
| High Quality Mode | Toggle | Warning: "can take 3x longer to generate" |
| Parse Script | Button | Triggers AI-based script parsing and video generation |

**Behavior Notes:**
- The system parses the script to extract discrete shots/scenes
- Each extracted shot is individually queued for video generation via Sora
- High Quality Mode warning is displayed to manage user expectations on wait time

---

### 5.3 History — Video Library

**Purpose:** Browse and review all generated videos, both personal and community-wide.

**Key UI Elements:**

| Element | Type | Description |
|---|---|---|
| Your Videos tab | Button | Filtered view showing only the current user's generated videos |
| All Videos tab | Button | Shows all videos generated by all users (4,370+ at time of exploration) |
| Video Count | Text | Shows total count of visible videos |
| Refresh button | Button | Manually reload the video list |
| Cookie Debug Panel | Collapsible panel | Shows the user's browser cookie UUID and current filter |

**Video Card Information (per entry):**
- Status badge (e.g., `Completed`)
- Generation duration (e.g., `2m 27s`)
- Timestamp (e.g., `3m ago`)
- Original prompt text
- Aspect Ratio (Portrait / Landscape)
- Video Duration (e.g., `12s`)
- Quality Mode (Standard / High Quality)
- Cookie ID (user identifier)

**User Identity Model:**
- Users are identified by a UUID stored in the browser cookie (e.g., `0dd2dddf-435b-484b-9f67-fa386e6ad3ab`)
- No login/authentication is required
- "Your Videos" is scoped to the current browser's cookie ID

## 6. Global Parameters (Available in Both Studio & Stock Footage)

| Parameter | Options | Default |
|---|---|---|
| Video Duration | 4s, 8s, 12s | 4s |
| Aspect Ratio | Landscape, Portrait | Landscape (Studio) / Portrait (Stock) |
| High Quality Mode | On / Off | Off |

---

## 7. Constraints & Edge Cases

- **Anonymous identity:** Clearing cookies would dissociate the user from their video history
- **Generation time:** Standard mode is significantly faster; High Quality can take ~3x longer
- **Simultaneous videos:** Studio supports up to 5 videos per single prompt submission
- **Unmodified prompts:** Studio is explicitly a "raw" interface — no AI prompt enhancement
- **Community history:** All Videos tab is public — all user-generated videos are visible to anyone
